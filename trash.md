### 以 Visual Studio Code (VS Code) 为例

**1. 在IDE中，将自己的Lab1纳入Git管理**
*   在 VS Code 中打开你的 Lab1 文件夹。
*   点击左侧活动栏的 **“源代码管理”** 图标（图标看起来像一个分叉的树枝，快捷键 `Ctrl + Shift + G`）。
*   点击面板上的 **`初始化仓库 (Initialize Repository)`** 按钮。

**2. 对Lab1进行若干修改，对其进行本地仓库提交操作**
*   在编辑器中修改文件并保存。
*   回到 **“源代码管理”** 面板，你会看到“更改 (Changes)”列表里有你修改的文件。
*   鼠标悬停在文件或“更改”栏上，点击出现的 **`+`** 号（暂存更改，相当于 `git add`）。文件会移动到“暂存的更改 (Staged Changes)”列表中。
*   在面板顶部的消息输入框中，输入提交说明（例如："在VS Code中修改并提交"）。
*   点击输入框下方的 **`提交 (Commit)`** 按钮（或输入框右侧的 `✓` 图标）完成本地提交。

**3. 将Lab1内容推送至个人GitHub的Lab1仓库**
*   点击“源代码管理”面板右上角的 **`...`**（视图和更多操作）图标。
*   选择 **`远程 (Remote)`** -> **`添加远程存储库... (Add Remote...)`**。
*   在顶部弹出的输入框中粘贴你的 GitHub 仓库 URL，按下回车。
*   接着输入远程仓库的名称（通常输入 `origin`），按下回车确认。
*   最后，点击面板上的 **`同步更改 (Sync Changes)`** 按钮，或者点击 **`...`** -> **`拉取、推送 (Pull, Push)`** -> **`推送 (Push)`**，将代码推送到 GitHub。
